Why the camel has six humps

The six structural pillars

The hump count is not decoration. It encodes the minimum stable abstractions that make this project usable and shippable: AMS orchestration, model runtime integration, repository contract, execution bus, operator surfaces, and GitHub release plane.

Cross-cutting concerns like security, monitoring, and analytics reinforce every hump, but they are not independent pillars and do not get separate humps.
